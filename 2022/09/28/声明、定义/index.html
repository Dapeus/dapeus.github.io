声明、定义

Dapeus
2022-09-28
C/C++

| 本文主要阐述这个问题，int a这个语句是声明还是定义？

其他简单的声明定义的区分可以参考这个文章

开始，例一

想要弄懂这个，需要懂一点编译的知识，在编译器预处理、编译、汇编之后，会生成一个ELF（可执行可链接文件，excutable linkable file），其中包含一个符号表symbol table，这个里面包含了这个C文件的所有定义的symbol，我简单的编译一个文件，讲解一下这个过程。

例如：

使用gcc -c test.c -o test.o之后，只进行到汇编，不链接，生成test.o，可以用file命令查看这个文件，file test.o ，可以看到是一个ELF file。

接着回到正题，我们要看它的符号表，使用readelf -s test.o查看符号表，如下

Num为9，10，11的行，可以看到三个符号都在其中，value是从第几个byte开始，size是占多少bytes，不用关心。

symbol type bind vis index
a OBJECT GLOBAL DEFAULT COMMON
b OBJECT GLOBAL DEFAULT 2
sum FUNC GLOBAL DEFAULT 1

解释一下，OBJECT表明这个symbol是一个数据对象（见下方macro definition），FUNC表明是一个代码对象，表明这个符号的类型，vis则不用例会，index是这个符号位于哪个section header中有一个关键的就是COMMON，这个COMMON表明这个符号只是一个声明，没有定义，后序根据其他文件中是否有这个符号的定义来决定它位于哪个section header，一般来说1就是.text section，2则是.data section，当然还有其他例如.bss，.rodata，.symtab等section。

COMMON的符号在链接阶段处理流程如下：如果其他文件中有这个符号的定义，则根据那个符号的index来决定它所在的index，如果有两个以上的定义，就报错，如果未找到，则会把它放入.bss section，未初始化数据段。

那么文章开头提出的问题，其实已经有答案了，int a其实就是一个定义语句。既然int a这个语句，会将a这个symbol作为COMMON处理，那么无论其它文件中是否出现了a，a也只会在.data section和.bss section中徘徊，都是属于定义的情况，我会在例子2，例子3中分别验证出现在.data section和.bss section的情况，如果不关心的细节，其实记住这个结论就可以了。

可以去/usr/include/elf.h中找到这些字段的定义，例如type和bind的取值范围，

type的取值，STT：symbol table type

bind的取值，STB：symbol table bind
